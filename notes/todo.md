# Технический долг и улучшения

## Editor Layout

- [ ] Оптимизировать триггер сохранения layout. Сейчас срабатывает на каждый пиксель
      через `dockviewApi.onDidLayoutChange`. Нужно придумать алгоритм который
      срабатывает только после завершения действия (конец resize, конец drag).

- [ ] Загрузка сохраненного layout - при десериализации params.component теряется
      при JSON.stringify. Сейчас восстанавливаем через реестр компонентов в
      EditorLayoutService.restore(). Проверить edge cases когда компонент не найден
      в реестре.

- [ ] EditorLayoutService.applyDefault() не реализован - добавить метод который
      восстанавливает дефолтный layout из конфига.

## WebSocket

- [ ] StrictMode отключен в main.tsx из-за проблем с двойным монтированием WsService.
      Вернуть StrictMode и сделать cleanup корректным через useEffect return.

- [ ] Нет переподключения при разрыве соединения. Добавить автоматический reconnect
      с exponential backoff в WsService.

## Auth

- [ ] Нет refresh token механизма. При истечении JWT пользователь просто получает
      401 и не понимает что произошло. Нужен либо refresh endpoint либо редирект
      на страницу логина с сообщением.

- [ ] Нет logout endpoint на бекенде. Сейчас logout только client-side через
      clearUser(). Добавить серверный logout с инвалидацией токена если понадобится.

## Валидация

- [ ] Нет валидации request body на бекенде (auth, projects, collaboration routes).
      Добавить JSON Schema валидацию через Fastify.

- [ ] Нет валидации environment variables при старте сервера. Если не задан
      JWT_SECRET - сервер стартует с дефолтным значением что небезопасно.

## CORS

- [ ] CORS настроен с origin: true - разрешает все домены. Ограничить до конкретных
      origins для production.

## ProjectDbManager

- [ ] Нет LRU eviction - если проектов станет очень много пул будет расти бесконечно.
      Добавить максимальный размер пула и вытеснение давно неиспользуемых подключений.

- [ ] Нет обработки случая когда директория проекта не существует при вызове get().
      Сейчас упадет с ошибкой файловой системы.

## Notifications

- [ ] Настройки уведомлений не загружаются из БД при открытии проекта. Сейчас
      всегда применяются дефолтные значения при перезагрузке.

- [ ] Нет анимации исчезновения уведомлений - только появление.

## UI

- [ ] Кнопка "Message" в CollaborativeSettings задизейблена - реализовать
      личные сообщения между пользователями проекта.

- [ ] Нет индикации онлайн статуса участников проекта в CollaborativeSettings.
      Показывать кто сейчас в сети через RoomManager.getMembers().

- [ ] FileSystemPanel, HierarchyPanel, InspectorPanel - заглушки, нет реального
      контента.

## Backend

- [ ] `FileService.moveFile` содержит дебаг `console.log` - убрать перед релизом.

- [ ] `computeHash` в `MetaService` читает файл целиком в память через `fs.readFile`.
      Заменить на потоковое вычисление хеша для больших файлов (>100MB).

- [ ] `downloadFile` не требует auth токена в URL - токен передаётся через заголовок,
      но скачивание через прямую ссылку (например `<a href>`) не добавляет заголовки.
      Нужно реализовать временный download-токен или подписанный URL.

- [ ] `scanAndEnsureMeta` запускается при старте сервера и обходит все проекты синхронно.
      При большом количестве файлов это будет тормозить старт. Перенести в фоновую задачу.

## File Explorer

- [ ] Контекстное меню: `copy` и `paste` - только `console.log`, буфер обмена
      для файлов не реализован.

- [ ] Контекстное меню: `duplicate` - только `console.log`, дублирование файла
      не реализовано.

- [ ] `FileExplorerPanel` в tile-режиме: `onRenameConfirm` приводит `IconSpaceItem`
      к `TreeItem` через `as unknown as TreeItem` - хрупкий каст. Унифицировать
      типы или сделать отдельный обработчик.

- [ ] `FileExplorerPanel`: ошибки в `handleDrop`, `handleNativeDrop`,
      `handleRenameConfirm` тихо проглатываются. Добавить нотификации об ошибках
      через систему уведомлений.

- [ ] `useFileExplorer`: нет обработки ситуации когда WS-событие приходит для
      пути вне текущего `currentPath` в tile-режиме - дерево и тайлы могут
      разойтись.

## UI библиотека

- [ ] `@dnd-kit` зависимости больше не используются после перехода на нативный
      DnD - удалить из `package.json`.

- [ ] `TreeView`: состояние `expanded` хранится локально в `TreeNode`. При обновлении
      дерева извне (WS-событие, reload) узлы схлопываются. Нужно поднять состояние
      раскрытия наверх или персистить его.

- [ ] `IconTileSpace`/`IconRowSpace` не поддерживают мультиселект (Shift+Click,
      Ctrl+Click). Сейчас `selected` - массив, но логика выбора одиночная.
